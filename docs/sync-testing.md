## 테스트 케이스

### TC-1: 첫 로그인 → 데이터 동기화
- [ ] 앱 실행 후 설정 → ☁️ 클라우드 동기화 → "Google로 로그인" 클릭
- [ ] Google 계정 선택 후 로그인 성공
- [ ] 우측 상단에 초록색 ● 아이콘 표시
- [ ] 설정 페이지에 "✅ 연결됨" 및 이메일 표시

### TC-2: 맥 → 폰 책 동기화
1. 맥:
- [ ] 책 1권 업로드
- [ ] 5분 내로 다른 기기(동일 Google 계정 로그인)에서 책 자동 출현
2. 휴대폰 (또는 다른 브라우저):
- [ ] 같은 계정으로 로그인
- [ ] 책 목록에 업로드한 책 표시
- [ ] 책 열람 가능

### TC-3: 진행도 동기화
- [ ] 맥에서 책 열고 3문장 클릭 (진행도 변경)
- [ ] 휴대폰 새로고침 후 같은 책 열기
- [ ] 같은 챕터/스크롤 위치에서 시작
- [ ] 진행률 % 동일

### TC-4: 단어장 양방향
- [ ] 휴대폰에서 단어 2개 저장
- [ ] 맥에서 60초 내 단어장에 자동 출현 (폴링)
- [ ] 맥에서 단어 뜻 편집
- [ ] 휴대폰에서 수정된 뜻 확인

### TC-5: 큐 동기화
- [ ] 맥에서 문장 3개 "⏰ 나중에 공부" 저장
- [ ] 휴대폰 큐에서 동일 문장 3개 표시
- [ ] 휴대폰에서 1개 공부 완료
- [ ] 맥에서 큰 카운트 감소 확인

### TC-6: 오프라인 → 온라인
- [ ] 비행기 모드 켜기 (오프라인)
- [ ] 책 읽기, 단어 추가, 큐 저장 모두 정상 동작
- [ ] 비행기 모드 끄기 (온라인 복귀)
- [ ] 우측 상단 아이콘 회색 → 초록색 전환
- [ ] 오프라인 중 추가한 데이터가 클라우드에 업로드됨

### TC-7: 충돌 해결 (LWW)
1. 맥에서 책 A의 진행도를 챕터 3으로 변경
2. 잠시 후 휴대폰 (오프라인 상태로) 같은 책 진행도를 챕터 5로 변경
3. 휴대폰 온라인 복귀
- [ ] 충돌 발생하지 않음 (크래시 없음)
- [ ] updated_at이 최신인 쪽(휴대폰)의 진행도 유지

### TC-8: 책 삭제 (Soft Delete)
- [ ] 맥에서 책 삭제 (confirm 확인)
- [ ] 휴대폰에서 해당 책이 사라짐
- [ ] Supabase DB에서 해당 책 행의 `deleted_at`이 null 아님 확인

### TC-9: 로그아웃
- [ ] 설정 → 로그아웃 클릭
- [ ] 로컬 데이터(책, 단어장, 큐) 그대로 유지됨
- [ ] 동기화 아이콘 사라짐
- [ ] 다시 로그인하면 기존 데이터 유지 + 클라우드 머지

### TC-10: 미로그인 상태

- [ ] 단어 저장 → 정상
